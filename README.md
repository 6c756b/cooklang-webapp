# cooklang-webapp

A mobile-optimized web app for browsing and viewing [Cooklang](https://cooklang.org) `.cook` recipe files. Deployable as a static site on any web server or GitHub Pages â€” no backend required.

## Features

- Folder navigation with breadcrumb, mirroring your file structure
- Full-text search across recipe names, categories, and tags
- Tag cloud with frequency sorting
- Ingredient highlighting in steps: ingredients (red), cookware (blue italic), timers (green)
- Portion scaling Ă—0.5 / Ă—1 / Ă—2 / Ă—3 / Ă—4 â€” re-parses live via WASM
- Recipe images: place a `.jpg`/`.png`/`.webp` next to the `.cook` file with the same name
- YAML frontmatter support: `title`, `tags`, `servings`, `time`, `nutrition`, and custom fields
- PWA with offline support via Service Worker

## Supported `.cook` format

Standard Cooklang syntax with two metadata styles:

**Cooklang native:**
```
>> title: Pancakes
>> tags: breakfast, quick
>> servings: 4
```

**YAML frontmatter (Obsidian-compatible):**
```
---
title: Pancakes
tags: [breakfast, quick]
servings: 4
time: 10 min prep, 25 min total
nutrition: 320 kcal, 8g fat, 12g protein
---
```

## Getting started

**Prerequisites:** Node.js 18+

```bash
git clone https://github.com/your-username/cooklang-webapp
cd cooklang-webapp
npm install
```

Add your `.cook` files to `public/recipes/`, then:

```bash
npm run index   # generate public/recipes/index.json
npm run dev     # dev server at localhost:5173
```

## Deployment

### Any static host (subfolder-safe)

```bash
npm run build
# upload contents of dist/ to your web root
```

The built app uses relative paths and works in any subdirectory.

### GitHub Pages (two-repo setup)

Keep recipes and the app in separate repositories so recipe updates don't require a rebuild.

**App repo** â€” build once, commit `dist/` contents to the recipes repo root.  
**Recipes repo** â€” contains `.cook` files in `recipes/` alongside the built app files.

Copy `scripts/generate-index.mjs` to the root of the recipes repo. It has no dependencies and runs standalone:

```bash
# in the recipes repo
node generate-index.mjs   # renames umlaut filenames to ASCII, writes recipes/index.json
git add .
git commit -m "update recipes"
git push
```

Enable GitHub Pages in the recipes repo settings (source: root of `main` branch).

### Apache

The `public/recipes/.htaccess` sets the correct MIME type for `.cook` files and a short cache TTL for `index.json`. Upload it alongside the recipes.
