# cooklang-webapp

A mobile-optimized web app for browsing and displaying [Cooklang](https://cooklang.org) `.cook` recipe files. Designed for iPhone Safari, deployable as a static site on any web server.

> **Language:** The UI is currently in German. Localization support may be added in the future.

## Features

- Folder navigation with breadcrumb (mirrors your file structure)
- Full-text search across recipe names and tags
- Tag cloud with frequency sorting and collapse
- Ingredient highlighting in step text (red), cookware (blue italic), timers (green)
- Portion scaling (×0.5 / ×1 / ×2 / ×3 / ×4) — re-parses live
- Recipe images: place a `.jpg`/`.png`/`.webp` with the same filename next to the `.cook` file
- Metadata from `>> key: value` syntax and YAML-style `---` frontmatter blocks
- PWA with offline support (Service Worker via Workbox)

## Cooklang format supported

Standard `>> key: value` metadata and Obsidian-style `---` wrapper blocks:

```
---
>> title: My Recipe
>> servings: 4
>> tags: pasta, quick
---

Add @salt{} and @pepper{} to taste.
Cook for ~{10%minutes}.
```

## Project structure

```
public/
  recipes/           ← .cook files and images (synced separately)
    index.json       ← generated by npm run index
    easy-pancakes.cook
    Lunches/
      pasta-carbonara.cook
      stuffed-peppers.cook
scripts/
  generate-index.mjs ← scans recipes/, writes index.json
src/
  components/        ← RecipeList, RecipeView, IngredientList, StepList, ScalingControl
  hooks/             ← useRecipes, useRecipe
  lib/cooklang.ts    ← parser wrapper + frontmatter handling
  types/recipe.ts
```

## Setup

```bash
npm install
npm run dev        # dev server at localhost:5173
npm run build      # production build → dist/
npm run index      # regenerate public/recipes/index.json
```

## Possible sync workflow

Recipes live in `public/recipes/` and are synced independently from the app build:

1. Write `.cook` files in Obsidian (or other editor)
2. Run `npm run index` to regenerate `index.json`
3. Upload `public/recipes/` to your web server (FTP/rsync)

The app itself is built once and deployed separately:

```bash
npm run build
# upload dist/ to your web server root
```

### `.htaccess` for Apache

The `public/recipes/.htaccess` sets the correct MIME type for `.cook` files and a short cache TTL for `index.json`. Upload it alongside the recipes.
